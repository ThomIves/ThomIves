10-Minute NEOGOV Cram Cheat Sheet

The five load-bearing things to absorb if you only have 10 minutes before the next conversation.

1. The behavior-change framing is the gravitational center

The JD's phrase "changed behaviors and direct pipeline impact, NOT output volume" is what every strategic answer should anchor to. Translation: the system is judged by whether sellers act differently because of it — not by how many briefs we ship.

One-line answer for the "how do you measure success" question:

"Pre-registered measurement plan that ties artifact instrumentation through Outreach and Salesforce telemetry to causal-inference cohort design, with explicit kill-switch criteria. Layered: lead behavioral indicators (action sheet open rate, time-to-touch on recommended accounts) → cohort comparison → account-level lift. And the honesty layer — we tell the CFO what we can't prove."

2. The five-layer whiteboard architecture

Memorize the sketch:

  1. Source connectors / CDC — Salesforce, Outreach, Gong, 6Sense, Pursuit, Starbridge, Tableau
  2. Bronze landing zone — raw schemas preserved with lineage
  3. Silver conformed layer — revenue entity model (Account, Opportunity, Contact, Engagement, Signal)
  4. Gold marts — one purpose-built mart per artifact (CRO / Director / seller / Marketing)
  5. AI intelligence layer — reads from gold, retrieves from Gong vectors, runs structured-output LLM passes, writes to delivery layer

Plus: orchestration plane (Dagster/Airflow/Fabric), eval harness beside the LLM, HITL checkpoints on high-stakes artifacts, monthly signal pack as its own batch pipeline.

3. Shared substrate + per-artifact composers

For "how do you produce four different artifacts from the same data without four pipelines or one mega-prompt":

"Shared deterministic substrate does the heavy lifting once — pull Salesforce snapshot, classify Gong calls, join 6Sense, produce a canonical opportunity-period fact table. Then four per-artifact composers each take their slice with audience-specific prompts and structured-output schemas. Numbers come from SQL, never from the model. Model only writes the connective tissue."

4. The hallucination containment layered defense for CRO brief

If the CRO acts on a hallucinated number, it's career-ending. Layered defense:

  1. Numerics pulled deterministically from SQL — never LLM-generated
  2. Every claim cites a source ID (Salesforce opp ID, Gong call ID, signal row ID)
  3. Cross-checking agent verifies before publication
  4. Generate Friday → human review Friday PM → ship Monday
  5. Provenance footnotes so the CRO can click through to source

5. Your strongest proof points

Lead with these in any applied-LLM answer:

  • Microsoft Fabric AI Functions classifier — gpt-4.1-mini at million-row scale with full evaluation scaffolding, structured-output validation, per-row cost / latency management. This is the closest direct analog to NEOGOV's daily work.
  • PostalDataPI — agent-native MCP API in 246 countries, single-handed concept to production. This is the MCP credential.
  • Echo Outlook AI Add-Ins — 30K+ daily requests, 5+ min → <15s cycle time, sales-rep and carrier sides. This is the marketplace-AI-in-the-tool-the-user-already-uses pattern.

What to downplay

  • Don't lead with framework names (LangGraph vs CrewAI) — junior tell
  • Don't lead with model preferences (GPT vs Claude) — junior tell
  • Don't lead with "I'd RAG it" for memory questions — junior tell
  • Don't pretend public-sector experience you don't have

Posture reminders

  • Listen first. Ask clarifying questions before proposing solutions for vague technical questions.
  • Anchor every technical answer in a specific past project — moves the answer from theoretical to demonstrated.
  • The behavior-change metric is the gravitational center. Pull strategic answers back to it.

You've got this.
